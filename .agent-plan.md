Architectural invariants worth remembering when editing:
- `SceneMixer.mix_sequential()` takes `(wav_bytes, amount_s, speaker_id, rms_target_dbfs, mix_mode)` 5-tuples; `MixMode` lives in `synthbanshee/tts/mix_mode.py` (no audio deps)
- `MixedScene` carries `script_*` / `rendered_*` / `audible_*` per-turn timelines; the label generator uses `audible_*` exclusively; `truncated: true` flags BARGE_IN-interrupted turns
- Temp WAV between mixer and `preprocess()` is `subtype="FLOAT"`; `preprocess()` uses `_peak_limit()` (limiter, not normalizer); `validate_audio()` rejects peak > −1.0 dBFS + 0.5 dB
- Loudness contract (#78): single global gain to `target_peak_dbfs` (default −2.0 dBFS) before the −1.0 dBFS safety limiter; preserves M3a per-turn RMS contrast
- Effective-prosody cap (#87): `_apply_effective_prosody_cap` in `tts/renderer.py` clamps post-state, post-randomization prosody to pitch ∈ [−3.0, +2.0] st, rate ∈ [0.85, 1.20]; cap activations recorded per-turn in `DialogueTurn.effective_prosody_caps`

## CI / Workflow notes

- `pr-agent-context` reusable workflow is referenced as `@v4` (floating major tag) in both `.github/workflows/ci.yml` and `.github/workflows/pr-agent-context-refresh.yml`. **Do not pin to a patch version.** The `tool_ref: v4` input mirrors it.
- The refresh workflow runs on a 15-minute `schedule` (dispatched via `workflow_dispatch` to each open same-repo PR) as a fallback for bot-authored review events.
- Tier-3 ASR sanity (`synthbanshee qa-report --asr`) is local-only per CLAUDE.md; runs once after fixes for any PR touching `synthbanshee/tts/`, `synthbanshee/script/`, `synthbanshee/augment/`, or speaker / scene / acoustic / project YAMLs.
